Go Testing

Unlike other languages, Go provides the testing framework in its standard library making testing in Go much simpler and indeed a must in anything going to production.

Writing tests are easy as Go coding in general. Lets see a simple test that checks a greeting function produces the correct output.

Writing a Test

package main

import "testing"

func TestSimpleGreeting(t *testing.T){

    got := greetMe("Keir")
    want := "Welcome, Keir"

    if got != want {
        t.Errorf("wanted %s, but got %s", want, got)
    }
}

The testing package is imported from the standard library, providing a plethora of methods we can use.

Tests in Go must start with the work “Test” with the first letter capitalised. Otherwise the test will not run when called.

The test needs to take in a pointer to the testing method T, which is commonly assigned to the variable t.

Save the file with the suffix _’test.go’ for example: greetMe_test.go - this is so the Go testing tool knows that this Go file contains tests it should be compiling and running.

Running the Go testing tool

$ go test *.go
# command-line-arguments [command-line-arguments.test]
./greetMe_test.go:7:9: undefined: greetMe
